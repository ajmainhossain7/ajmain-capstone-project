# CLAUDE.md

## Project Stack

- Node.js with npm
- Git for version control
- Markdown for project documentation

## Coding Conventions

- Keep code simple, readable, and well-organized.
- Use descriptive names for files, functions, and variables.
- Prefer consistent indentation and formatting.
- Avoid adding application code before the initial project setup is complete.

## Git Commit Conventions

- Use Conventional Commits.
- Format commits as `type: short summary`.
- Example: `feat: add user authentication`.

## Project Goals

- Establish a clean foundation for the capstone project.
- Keep documentation and repository structure professional and maintainable.
- Prepare the project for future feature development.
